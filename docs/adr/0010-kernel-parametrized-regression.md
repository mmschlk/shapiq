# Kernel-Parametrized Regression Estimation

The regression family opens from a closed index union to the regression capability itself, completing for the argmin primitive what ADR 0009 did for linear functionals. ``Regression`` now builds its sampler from the index's declared kernel: coalition sizes are drawn with weight ``kernel(size) * comb(n, size)``, members uniform within a size, so every coalition appears with probability proportional to its kernel weight and every sampled row enters the least squares problem with unit weight — the invariant ADR 0007 identified as the reason the union had to stay closed until the sampler became kernel-parametrized. The kernel contract is validated where the sampler is derived: one nonnegative finite weight per size, with the empty and grand coalition at exactly zero because they are interpolated as constraints, never weighted. ``define_regression_index`` declares a new index from a kernel function alone, and both the exact solver (which was already capability-generic) and the sampled estimator come with the declaration.

Pairing interacts with the derived proposal: emitting every coalition together with its complement preserves the proportional-to-kernel invariant only when the kernel is size-symmetric. Symmetrizing the proposal silently — which is what ``CoalitionSizeSampler`` does for the importance-weighted ``MonteCarlo``, where the estimator divides by the actual sampling probability — would silently corrupt the unit-weight regression instead, so ``Regression`` rejects ``paired=True`` for asymmetric kernels with an error naming the fix. This is the general shape of the difference between the two derived estimators: ``MonteCarlo`` reweights per sample and tolerates any fully-supported proposal, while ``Regression`` encodes the kernel in the sampling distribution and therefore must control it exactly.

``ShapleyKernelSampler`` is deleted rather than kept as an alias: the Shapley kernel size distribution ``1/(t(n-t))`` is exactly the normalized ``kernel * comb`` profile of the Shapley kernel, and the derived sampler reproduces the same evidence stream draw for draw, so a dedicated class described nothing anymore. One grammar beats two, as ADR 0007 put it about the class-per-index names.

kADD-SHAP gains the sampled estimator that ADR 0008 anticipated when it kept the index declaring the Shapley regression kernel. Its basis is dispatched, not generalized away: ``Regression`` fits the membership basis with the all-ones constraint substitution for every constrained-interpolation index, and the Bernoulli basis with general pivot elimination for kADD-SHAP, both sharing the pinned-constraint solver (``solve_pinned``) with the exact explainer. FBII stays outside the family: its fit is unconstrained with a free intercept, which contradicts the capability's constrained-interpolation contract, so an unconstrained regression variant remains future work rather than a silent special case.
